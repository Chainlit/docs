### Options

<ParamField path="prompt_playground" type="bool" default={true}>
  Display the [prompt playground](/advanced-features/prompt-playground) when
  passing a prompt to a message.
</ParamField>

<ParamField path="unsafe_allow_html" type="bool" default={false}>
  Process and display HTML in messages. This can be a security risk (see
  https://stackoverflow.com/questions/19603097/why-is-it-dangerous-to-render-user-generated-html-or-javascript).
</ParamField>

<ParamField path="latex" type="bool" default={false}>
  Process and display mathematical expressions. This can clash with "$"
  characters in messages.
</ParamField>

<ParamField path="multi_modal.enabled" type="bool" default={true}>
  Authorize users to upload files with messages. The files are then accessible
  in [cl.on_message](/api-reference/lifecycle-hooks/on-message).
</ParamField>

<ParamField path="multi_modal.accept" type="Union[List[str], Dict[str, List[str]]]" default={["*/*"]}>
  Restrict user to only upload accepted file types.
</ParamField>

<ParamField path="multi_modal.max_files" type="int" default={20}>
  Restrict user to upload maximum number of files at a time. 
</ParamField>

<ParamField path="multi_modal.max_size_mb" type="int" default={500}>
  Restrict uploading file size (MB). 
</ParamField>

<ParamField path="speech_to_text.enabled" type="bool" default={false}>
  Allows user to write messages with speech.
</ParamField>

<ParamField path="speech_to_text.language" type="string">
  The language spoken by the user. Greatly improve the quality of the
  transcription. See available languages here
  https://github.com/JamesBrill/react-speech-recognition/blob/HEAD/docs/API.md#language-string
</ParamField>

### Default configuration

```toml
[features]
prompt_playground = true
unsafe_allow_html = false
latex = false
[features.multi_modal]
    enabled = true
    accept = ["*/*"]
    max_files = 20
    max_size_mb = 500
# Allows user to use speech to text
[features.speech_to_text]
    enabled = false
    # language = "en-US"
```
